# 方舟参考素材视频生成 Composer：开发者交接说明

本文是中文开发者接手该原型时的入口文档。需要快速运行、确认边界、继续迁移或做二次开发时，优先读本文，再按需跳转到 README、迁移指南和 QA 报告。

## 1. 一句话说明

该项目复刻的是方舟控制台视频生成输入组件里的“参考素材 + `@图片N` 内联引用 + Dry-run 请求预览”能力。它不是方舟整站 clone，也不是付费生成自动化脚本。

## 2. 本地运行

```bash
npm install
npm run serve
```

默认地址：

```text
http://127.0.0.1:8080/#/video-generation
```

端口冲突时：

```bash
VITE_CLI_PORT=43127 VITE_SERVER_PORT=43128 npm run serve
```

健康检查：

```bash
curl http://127.0.0.1:43128/api/health
```

期望返回：

```json
{ "code": 0, "data": { "status": "ok" }, "msg": "服务正常" }
```

## 4. 关键数据流

```text
用户上传图片
  ↓
POST /videoGeneration/uploadReference
  ↓
服务端返回素材 ID、previewUrl、name、mimeType、size
  ↓
Pinia store 分配稳定 realIndex
  ↓
编辑器 @ 菜单插入 mediaMention 原子节点
  ↓
POST /videoGeneration/dryRun
  ↓
服务端重新读取权威素材元数据
  ↓
返回 serialization + Ark request + blockers
```

前端提交给服务端的素材信息只包含：

```js
{ id: '素材 ID', realIndex: 1 }
```

不要让前端提交的 `remoteUrl` 或 `previewUrl` 参与真实生成决策。

## 5. `@图片N` 不应当如何实现

不要这样做：

- 不要在 textarea 字符串里手动拼 `@图片1`。
- 不要用正则从文本里解析素材。
- 不要让 `图片1` 的编号随删除重新编号。
- 不要把本地预览 URL 直接发给 Ark。

正确方式：

- Tiptap 文档中插入 `mediaMention` 原子节点。
- 原子节点只保存素材 ID 和展示辅助信息。
- 提交时服务端重新解析素材 ID。
- 序列化层生成可读提示词、模板提示词、模型提示词和 `image_url`。

## 6. Dry-run 与真实生成

默认提交只走 Dry-run。Dry-run 是解释和校验请求结构的工具，不创建付费任务。

真实生成必须满足：

- `APP_REAL_GENERATION_ENABLED=true`
- `ARK_API_KEY` 在服务端存在
- 素材 URL 对 Ark 可访问
- 当前草稿 Dry-run 成功
- `confirmationToken` 有效且未被消费
- 用户在预览抽屉中显式确认

实现中不做自动重试，原因是视频生成任务是潜在付费副作用，网络错误下无法安全判断是否已经创建成功。

## 7. 本地浏览器 QA 的特殊入口

`src/view/videoGeneration/index.vue` 支持 development-only URL 参数：

```text
#/video-generation?qaMedia=<encoded-json>
```

用途：当浏览器扩展没有本地文件上传权限时，先通过本地 API 上传图片，再把服务端返回的素材元数据 seed 到页面，用于复现 `@` 菜单和 Dry-run 链路。

约束：

- 只在 `import.meta.env.MODE === 'development'` 时生效。
- 生产构建不会用它做业务入口。
- 不用于绕过服务端校验；真实 Dry-run 仍会走服务端权威素材解析。

## 8. 迁移到 hc-gpt-web 时最容易踩的点

1. `src/api/videoGeneration.js` 可以搬，但不要搬原型的 `src/utils/request.js`。
2. `requestBuilder.js` 是纯函数，建议保留测试一起迁移。
3. `server/media/store.js` 是本地适配实现，迁移时要替换成正式上传/CDN/对象存储。
4. `confirmationStore.js` 可以换成 Redis 或数据库，但必须保持一次性 token 语义。
5. `createTask` 不要自动重试。
6. 真实生成前一定要先在 UI 上展示最终请求预览和阻塞项。

## 10. 后续建议

- 接入正式素材 CDN 后，先只做 Dry-run，确认 `MEDIA_NOT_PUBLIC` blocker 消失。
- 若确需真实生成验证，使用最低成本参数：`720P / 5秒 / 1条 / 无声`，并在操作时再次确认。
- 若迁移到 `hc-gpt-web`，建议先以隐藏菜单或内部路由灰度，不直接开放给普通用户。
