# Security Policy

## Reporting a vulnerability

Please use GitHub Private Vulnerability Reporting for this repository. Do not publish credentials, private media, complete real task IDs, temporary result URLs, or exploit details in a public issue.

Include affected versions, reproduction steps, impact, and a proposed mitigation when possible. Maintainers will acknowledge a report through the private channel, assess severity, coordinate a fix, and disclose it after users have a reasonable opportunity to update.

## Supported scope

This repository is a learning and migration reference, not a production service. Deployers are responsible for authentication, authorization, persistent storage, object storage, rate limits, abuse prevention, audit logging, secrets management, and compliance appropriate to their environment.
